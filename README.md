# 제목 : 비트코인 결제와 기존 금융 지급결제의 구조적 차이 비교

## 부서명 : 여신서비스개발부

## 이름 : 김동욱

## 레포트 내용

비트코인의 지급결제 구조를 우선 파악하며 기존 금융 지급결제 구조와 어떤 차이가 있는지 서술해보도록 하겠다.

비트코인의 지급결제 구조는 중앙은행이나 금융기관과 같은 중개자 없이, \*P2P(Peer-to-Peer) 네트워크를 통해 참여자들이 직접 거래를 검증하고 기록하는 탈중앙화된 시스템이다.

은행(기존금융)의 결제 시스템과 가장 큰 차이점은 거래를 하고 난 후 계좌의 잔고를 Update 하는 '계좌 잔고' 모델이 아닌 \*UTXO라는 고유한 방식을 사용하며, 모든 거래 내역이 분산 원장인 블록체인에 기록된다는 점이다.

비트코인의 지급결제는 크게 다음의 5단계를 거쳐 이루어진다.

1. 거래 생성 및 서명

송금자는 비트코인 지갑을 이용해 수신자의 비트코인 주소(공개키 기반)로 코인을 보내는 거래(Transaction)를 생성한다. 이때 송금자는 자신이 해당 비트코인의 정당한 소유자임을 증명하기 위해 자신의 개인키를 사용하여 거래 내역에 암호학적 디지털 서명을 남긴다.

2. 네트워크 전파

서명된 거래 데이터는 인터넷을 통해 비트코인 네트워크에 연결된 수많은 노드(Node, 네트워크 참여 컴퓨터)들에게 정보가 전파된다.

3. 검증 및 멤풀 대기

새로운 거래를 전달받은 노드들은 해당 거래가 규칙에 맞는지 즉각 검증한다. 서명이 유효한지, 지불하려는 비트코인(UTXO)이 실제로 존재하고 아직 쓰이지 않았는지(이중 지불 방지) 등을 확인한다. 이 검증을 무사히 통과한 거래들은 아직 장부에 기록되지 않은 대기 상태로 \*멤풀(Mempool, Memory Pool)이라는 임시 공간에 모이게 된다.

4. 작업 증명과 블록 생성

채굴자들은 멤풀에서 대기 중인 거래들을 모아 하나의 새로운 블록 후보를 구성한다. 보통 채굴자들은 수익을 극대화하기 위해 거래 수수료가 높게 설정된 거래를 우선적으로 블록에 포함시킨다.블록을 체인에 연결할 권리를 얻기 위해 채굴자들은 컴퓨팅 파워를 동원하여 목표값보다 작은 해시 값을 찾는 경쟁을 하는데, 이를 작업 증명(PoW)이라고 한다.

5. 블록 전파 및 최종 확정

가장 먼저 해시값을 찾은 채굴자는 자신이 만든 블록을 네트워크에 전파한다. 다른 노드들이 이 새 블록의 유효성을 검증하고 이상이 없음을 동의(합의)하면, 기존 블록체인의 맨 끝에 이 블록이 추가된다. 내 거래가 포함된 블록이 체인에 연결되는 순간 1차 승인(1 Confirmation)이 발생한다. 비트코인은 대략 10분마다 새로운 블록이 생성되는데, 내 거래가 담긴 블록 뒤로 새로운 블록들이 계속 이어져 붙을 때마다 승인 횟수가 올라간다. 일반적으로 해킹이나 거래 취소가 수학적으로 불가능해지는 6번의 승인(대략 60분)이 이루어지면, 지급결제가 최종적으로 확정된 것으로 간주한다.

짧게 정리하자면, 비트코인의 결제 구조는 개인키 서명을 통한 지불 요청 -> 네트워크 노드들의 분산 검증 -> 채굴자들의 작업 증명을 통한 장부기록(블록체인)이다. 기존 금융에서는 계좌 자체에 대한 정보를 더욱 중요도 있게 여기고 계좌 잔고를 update하면서 직관적으로 가지고 있는 구조라면, 비트코인(블록체인)은 수많은 UTXO의 결과물의 총합을 보여줌으로써 계좌의 잔액을 보여주는것이고 실제 신뢰성 기반으로 관리되는것은 UTXO들이라고 볼 수 있다.

**UTXO(미지출 거래 출력값, Unspent Transaction Output)란?**

UTXO는 비트코인(블록체인) 네트워크에서 '잔고'를 인식하고 거래를 처리하는 핵심 아키텍처다.

기존 금융권의 계정계 원장에서 사용하는 '계좌 잔고' 아키텍쳐와 비교하면 그 차이를 직관적으로 이해할 수 있다.

1. 계좌 잔고 vs UTXO

기존 금융의 지급 결제에 대한 관리는 데이터베이스에 특정 사용자의 계좌 식별자(계좌번호)와 현재 잔고(잔액)라는 값이 저장되어 있다. 송금이 발생하면 시스템은 송금자의 잔고가 충분한지 확인한 후 송금자의 잔고 상태 값을 마이너스(Update) 처리하고, 수신자의 잔고(잔액) 값을 플러스 처리한다.

비트코인 시스템에는 '계좌'나 '현재 잔액'이라는 데이터베이스 필드 자체가 아예 존재하지 않는다. 오직 '아직 쓰이지 않은 거래의 결과물(UTXO)'들만 네트워크 상에 흩어져 있다. 누군가의 '지갑 잔고'라는 것은, 그 지갑의 공개키(주소)로 묶여 있는 수많은 UTXO들의 총합을 지갑 소프트웨어가 계산해서 화면에 보여주는 것일 뿐이다.

2. UTXO는 '실물 지폐'와 같습니다.

UTXO는 쪼갤 수 없는 실물 지폐나 동전과 정확히 같은 방식으로 동작한다.

예를 들어, 지갑에 5 BTC짜리 UTXO 1개와 3 BTC짜리 UTXO 1개가 있다고 가정하자. (화면상 잔고는 8 BTC로 보인다.) 여기서 다른 사람에게 2 BTC를 보낸다고 한다면

Input : 은행 계좌처럼 8 BTC 중 2 BTC만 쏙 빼서 보낼 수는 없다. 3 BTC짜리 UTXO 1개를 통째로 이번 거래의 '입력값'으로 사용한다. 이때 사용된 3 BTC짜리 UTXO는 이제 '지출됨(Spent)' 상태가 되어 소멸한다.

Output : 거래의 결과로 2개의 새로운 UTXO가 생성된다.

수신자에게 지급되는 2 BTC짜리 UTXO (수신자의 공개키로 암호화)
거스름돈으로 나에게 돌아오는 0.999 BTC짜리 UTXO (나의 공개키로 암호하가 된다. 0.001 BTC는 채굴자 수수료)
결과적으로 하나의 거래는 반드시 기존의 UTXO를 파괴(Consume)하고, 새로운 UTXO를 창조(Create)하는 과정을 거친다.

3. 왜 비트코인은 이런 복잡한 방식을 쓸까요?

개발 관점에서 UTXO 모델은 상태 비저장 아키텍처로 본다. 이로 인해 얻는 이점들이 다음과 같다.

이중 지불 방지의 명확성: 각 통화 단위(UTXO)가 독립적인 객체이므로, 네트워크 노드들은 특정 UTXO가 이미 지출되었는지만 확인하면 된다. 상태 업데이트 과정에서 발생할 수 있는 트랜잭션 충돌이나 동기화 이슈를 줄일 수 있다.

병렬 처리의 용이성 : 각기 다른 UTXO를 사용하는 거래라면, 서로 간섭하지 않으므로 논리적으로 병렬 검증 및 처리가 가능하다.

프라이버시 강화 : 거스름돈을 받을 때마다 매번 새로운 주소(공개키 해시)를 생성하여 새로운 UTXO를 할당받을 수 있습니다. 이렇게 하면 거래 내역을 추적하여 특정 개인의 자산 규모를 특정하기가 매우 까다로워진다.

**비트코인(블록체인)은 언제 지급 결제과 '완료' 혹은 '확정' 되었다고 얘기할 수 있을까?**

은행의 계좌 잔고 모델은 하나의 거래 트랜잭션이 수행되고 데이터베이스의 계좌의 잔고가 update 되었다면 거래가 완료 되었고 확정되었다고 볼 수 있을 것이다. 하지만 비트코인은 장부기반이기에 장부의 확정 순간을 봐야 할 것이다.

1. UTXO와 트랜잭션의 관계

트랜잭션은 어떠한 지불결제의 행위 라고 본다면, UTXO는 그 행위가 끝나고 난뒤 나온 결과물이라고 보아야 할 것 같다.

2. 멤풀에는 무엇이 들어있을까?

멤풀 안에 UTXO가 직접 들어가 있는것이 아니라, 멤풀에는 '아직 검증되지 않은 트랜잭션(행위)'들이 쌓여 있다. 사용자가 서명한 트랜잭션이 멤풀에 도달하면, 노드들은 "이 트랜잭션이 사용하려는 UTXO가 정말로 존재하고 아직 안 쓰인 것인가?"를 확인한다. 즉, 멤풀은 "새로운 UTXO를 만들기 위해 대기 중인 트랜잭션들의 대기줄"이다.

3. 거래 확정의 시작: "블록 후보의 체인 연결"

멤풀에 있던 트랜잭션이 채굴자에 의해 선택되어 블록에 포함되고, 그 블록이 이전 블록 뒤에 연결되는 순간이 바로 거래 확정의 시작이다.

0 Confirmation: 멤풀에만 있는 상태. 언제든 네트워크에서 사라지거나 교체될 수 있어 확정이라 보기 어렵다.

1 Confirmation: 블록에 포함되어 체인에 연결된 직후. 이때부터 "거래가 장부에 적혔다"고 하며 확정이 시작된다. 하지만 비트코인 블록체인 네트워크에서는 가장 긴 노드만이 신뢰받기 때문에 100퍼센트 확정이라고 볼 수는 없다.

N Confirmations: 그 뒤로 블록이 계속 연결되는 과정. 이것이 진정한 의미의 '결제 완료'를 향해가는 과정이다.

정리하면 "멤풀 안에 트랜잭션(거래 요청)들이 대기하다가, 채굴자가 이를 모아 만든 블록이 체인에 연결되는 순간부터 새로운 UTXO가 생성되며 거래 확정이 시작된다." 라고 볼 수 있다.

일반적으로 비트코인 블록체인은 6번의 승인을 거래 확정으로 보고 있다. 비트코인 네트워크에서는 일시적으로 두 명의 채굴자가 동시에 블록을 생성하여 체인이 갈라지는 고아 현상이 발생할 수 있습니다. 하지만 확률적으로 6개의 블록(약 1시간)이 쌓이는 동안 한쪽 체인이 압도적으로 길어지게 되며, 네트워크는 '가장 긴 체인'을 가장 신뢰받는 장부로 선택한다.

6번의 승인이 완료된 거래를 뒤집으려면 전 세계 비트코인 연산력의 51% 이상을 확보해야 하는데, 이는 천문학적인 비용이 들기 때문에 사실상 경제적 최종성(Economic Finality)이 확보된 것으로 본다.

비트코인과 기존금융의 지급결제 구조를 거래의 과정과 확정의 순간을 중점으로 내용을 정리하자면, 기존 금융 시스템은 계좌에 대한 정보를 직관적으로 관리하며 트랜잭션단위로 거래를 확정 짓는다. 비트코인은 트랜잭션들을 모아 블록을 네트워크에 연결시키는 순간이 거래 확정의 시작이라고 볼 수 있고 수학적인 확률로 완전한 확정이라고 보기까지 기존 금융 시스템에 비해 처리 시간이 많이 느리다고 볼 수 있다.
